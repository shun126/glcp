# tests

`tests/` contains the smoke tests and runtime coverage tests for generated `glcp` / `glcp_compat` outputs.

## Directory layout

- `compile/`
  - Verifies that the headers can be included and that representative API symbols can be referenced at compile time.
  - Generated loader validation also checks that header declarations, source definitions, initialization assignments, finalize resets, and generated inventories stay in sync by version block.
- `common/`
  - Contains shared macros such as `TEST_ASSERT` and shared runtime coverage helpers.
  - `runtime_coverage.h` executes OpenGL APIs that can be called safely by category and checks that `glGetError()` does not report unexpected errors.
  - `core_runtime_coverage.h` and `compat_runtime_coverage.h` use function inventories generated by CMake to verify loaded function counts and NULL reset after finalize.
- `win/`
  - Creates a WGL context on Windows and runs the core / compat runtime tests.
- `macos/`
  - Creates a CGL context on macOS and runs the core / legacy runtime tests.
- `glx/`
  - Creates a GLX context on Linux and runs the core / compat runtime tests.

## What is tested

The current runtime tests create a real OpenGL context on each platform and verify the following:

1. Function pointers are loaded after `glcpInitialize()` or `glcpCompatInitialize()`.
2. The function inventory extracted by CMake from `glcp/glcp.c` or `glcp/glcp_compat.c` can be traversed safely.
3. Inventory functions are grouped by `GLCP_GL_VERSION_x_y`, and each generated loader version block up to the current runtime OpenGL version is checked for loaded / missing symbols after initialize.
4. Groups of APIs that can be called with safe minimal state are executed, and `glGetError()` returns `GL_NO_ERROR`.
5. Function pointers in the inventory are reset to `NULL` after `glcpFinalize()` or `glcpCompatFinalize()`.
6. Generated loader structure stays internally consistent:
   - `extern PFN... glXxx;` declarations in the header
   - `PFN... glXxx = NULL;` definitions in the source
   - `glXxx = (PFN...)glcpGetProcAddress("glXxx");` initialization assignments
   - `glXxx = NULL;` finalize resets
   - the CMake-generated function inventory used by runtime tests

The main API categories currently exercised are:

- Basic state operations
  - `glClearColor`, `glClear`, `glViewport`, `glScissor`, `glEnable`, `glDisable`, `glBlendFunc`, `glDepthFunc`, `glStencilFunc`, `glPixelStorei`, `glGet*`
- Texture
  - `glGenTextures`, `glBindTexture`, `glTexParameteri`, `glTexImage2D`, `glGetTexParameteriv`, `glDeleteTextures`
- Buffer
  - `glGenBuffers`, `glBindBuffer`, `glBufferData`, `glBufferSubData`, `glGetBufferParameteriv`, `glGetBufferSubData`, `glDeleteBuffers`
- Shader / Program
  - `glCreateShader`, `glShaderSource`, `glCompileShader`, `glGetShaderiv`, `glCreateProgram`, `glAttachShader`, `glLinkProgram`, `glGetProgramiv`, and delete calls
- Optional modern groups
  - VAO, FBO / RBO, query, sync, sampler, DSA buffer / texture
- Compat only
  - `glPushAttrib`, `glPopAttrib`, `glBegin`, `glVertex2f`, `glEnd`

Optional functions are skipped when the function pointer is `NULL`. This allows the test suite to continue even when a driver or platform profile does not expose a given API.

## How inventory generation works

`glcp_write_function_inventory()` in `CMakeLists.txt` reads `glcp/glcp.c` and `glcp/glcp_compat.c`, then generates `.inc` files in the build directory from assignment lines of the form `glcpGetProcAddress("glXxx")`.

- `glcp_core_function_inventory.inc`
- `glcp_compat_function_inventory.inc`

These `.inc` files are test-only function inventories, not distribution artifacts. They preserve the same `#if defined(GLCP_GL_VERSION_x_y)` guards as the generated headers, so the test code can reference functions safely under the same version macros.

`tests/validate_generated_loader.py` also parses the generated headers, sources, and inventory files directly and verifies that the function sets and prototype names match for every OpenGL version block that `glcp` emits.

At runtime, the platform tests also consume versioned inventory `.inc` files and fail when a generated loader version block that should be available for the active runtime OpenGL version reports missing symbols.

## When a new supported GL version is added

When support for a new OpenGL version is added, update and verify the test side as well:

1. Regenerate the outputs
   - Run `glcp.rb` and regenerate `glcp/glcp.c`, `glcp/glcp.h`, `glcp/glcp_compat.c`, and `glcp/glcp_compat.h`.
   - Confirm that a new `GLCP_GL_VERSION_x_y` block appears in `glcp/glcp.c`, and in `glcp/glcp_compat.c` when relevant.
2. Verify that inventory generation still matches
   - The regular expression in `CMakeLists.txt` assumes lines of the form `glXxx = (PFN...)glcpGetProcAddress("glXxx")`.
   - If the generated code format changes, update the extraction logic in `glcp_write_function_inventory()`.
3. Add representative APIs that can be called safely for the new version
   - Add new test helpers in `tests/common/runtime_coverage.h` under the appropriate version macro.
   - Prefer APIs that can be called with minimal state and do not depend heavily on external OS resources or vendor-specific behavior.
4. Confirm that `glGetError()` remains a valid acceptance check
   - New calls should use valid minimal arguments and minimal valid state, with `GL_NO_ERROR` as the expected result.
   - APIs that are inherently error-prone or require complex object setup should stay at load verification only instead of being forced into runtime coverage.
5. Decide whether the coverage belongs to core or compat
   - Core-only APIs should be added under the relevant `GLCP_GL_VERSION_x_y` guard.
   - Fixed-function behavior that only makes sense in compat should stay in `glcp_test_run_compat_fixed_function_calls()`.
6. Keep the execution path shared across platform runtime tests
   - In most cases there is no need to add more platform-specific test files under `tests/win`, `tests/macos`, or `tests/glx`.
   - Keep new coverage reachable through the shared helpers that the existing runtime tests already call.
7. Verify in CI
   - Linux: `xvfb-run -a ctest --test-dir build-tests --output-on-failure -C Release`
   - Windows / macOS: `ctest --test-dir build-tests --output-on-failure -C Release`

## What not to do

- Do not call every function indiscriminately with dummy arguments.
- Do not add APIs with heavy preconditions just to increase the coverage count.
- Do not add vendor extensions, external memory paths, special synchronization flows, or strongly GPU-dependent features to the shared tests without fallback or skip conditions.

The purpose of this directory is not to provide complete OpenGL semantic validation. It is to continuously verify that `glcp` exposes function pointers correctly and that representative APIs can be called safely.
